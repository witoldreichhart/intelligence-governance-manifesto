# Contributing to the Intelligence Governance Manifesto

This is a living framework. Intelligence governance is a new discipline — the principles will evolve as implementations test them.

## How to contribute

**Challenge a principle.** If a principle is wrong, impractical, or incomplete — open an issue explaining why and what you'd replace it with. The most useful challenges come with evidence from implementation experience.

**Suggest an addition.** If something critical is missing — a failure mode, a boundary condition, a worked example — open an issue or submit a pull request.

**Share implementation experience.** If you've implemented any of these principles in practice, we want to hear what worked, what didn't, and what the manifesto gets wrong. Open an issue tagged `implementation`.

**Propose a worked example.** The companion guide needs examples from different regulated domains (financial services, biotech, aviation, energy). If you have domain expertise, a worked example showing principles in action is extremely valuable.

## What we're not looking for

- Cosmetic edits or reformatting
- Adding principles without removing or restructuring existing ones (the twelve are deliberate)
- Vendor-specific implementation guidance (this manifesto is technology-agnostic)

## Code of conduct

Be rigorous. Be specific. Be constructive. Critique the ideas, not the people.
